Komisi

Model 1: Periklanan

Deskripsi

Aplikasi gratis, pendapatan dari iklan (tayangan/klik).

Contoh: TikTok

Mayoritas pendapatan dari in-feed ads dan branded content.

Kelebihan

  • Barrier rendah
  • Pertumbuhan cepat
  • Skalabilitas tinggi

Tantangan

  • Butuh user besar
  • Iklan ganggu UX
  • Persaingan tinggi

Model 2: Freemium

Deskripsi

Inti gratis, fitur premium berbayar. Tarik banyak user lalu konversi.

Contoh: Spotify

Gratis: iklan + batasan. Premium: tanpa iklan, audio HD, offline.

Kelebihan

  • Bangun loyalitas
  • Try before buy
  • Funnel jelas

Tantangan

  • Konversi 2-5%
  • Balance sulit
  • Butuh value kuat

Model 3: In-App Purchase

Deskripsi

Jual barang virtual/konten digital dalam app. Umum di game.

Contoh: Mobile Legends

Jual diamond & skin hero. Opsional, tingkatkan experience.

Kelebihan

  • Revenue dari "whale"
  • Pricing fleksibel
  • Berkelanjutan

Tantangan

  • Balance gameplay
  • Risiko pay-to-win
  • Regulasi loot box

Model 4: Subscription

Deskripsi

Bayar berkala untuk akses. Pendapatan stabil & predictable.

Contoh: Netflix, Spotify Premium

Langganan bulanan untuk unlimited konten.

Kelebihan

  • Pendapatan stabil
  • LTV tinggi
  • Cash flow konsisten

Tantangan

  • Beri nilai terus
  • Kelola churn
  • Banyak kompetitor

Model 5: Paid App

Deskripsi

Bayar sekali untuk download. Jarang untuk pasar massal.

Contoh: Procreate, Monument Valley

One-time payment, akses penuh, tanpa iklan.

Kelebihan

  • Pemasukan langsung
  • Tanpa iklan
  • UX fokus

Tantangan

  • Sulit dapat user
  • Value harus kuat
  • Alternatif gratis

Model 6: Marketplace Fee

Deskripsi

Platform pertemukan buyer-seller, ambil komisi.

Contoh: Gojek, Tokopedia, Airbnb

Perantara ambil % dari setiap transaksi.

Kelebihan

  • Skala tanpa inventory
  • Revenue per transaksi
  • Network effect

Tantangan

  • Chicken-egg problem
  • Trust dua sisi
  • CAC awal tinggi

Marketplace: Tantangan Utama

Masalah "Chicken and Egg"

Butuh banyak supply (driver/penjual) dan demand (user) bersamaan.

Solusi

  • Bangun ekosistem dua sisi seimbang
  • Insentif awal (subsidi, promo)
  • Menangkan trust kedua sisi
  • Platform jadi quality control

Developer (Pengembang)

Peran

Otak di balik aplikasi - dari individu hingga perusahaan besar.

Tanggung Jawab

  • Rancang dan kembangkan aplikasi
  • Maintenance dan update
  • Tentukan strategi monetisasi
  • Tarik dan retain pengguna
  • Optimasi user experience

User (Pengguna)

Peran

Nyawa ekosistem. Tanpa user, tidak ada ekonomi aplikasi.

Kontribusi

  • Gunakan aplikasi dan layanan
  • Hasilkan data perilaku
  • Beri feedback untuk perbaikan
  • Kontribusi metrik: DAU, MAU
Model Gratis: User "bayar" dengan data & perhatian → dimonetisasi via iklan.

Advertiser & Ad Network

Advertiser

Brand bayar tampilkan iklan. Contoh: Unilever, Samsung, Tokopedia.

Ad Network

Perantara cocokkan iklan + target. Contoh: AdMob, FB Audience Network.

Kolaborasi

Advertiser → audiens | Developer → revenue share | Otomatis skala besar.

Payment Gateway

Fungsi

Kasir digital fasilitasi transaksi aman user-aplikasi.

Pemain Indonesia

Midtrans, Xendit, Stripe, PayPal - Hubungkan ke bank, e-wallet, kartu.

Ambil fee per transaksi, tapi crucial: startup tak perlu bangun sistem payment sendiri.

Interaksi Antar Pemain

Rantai Nilai

  • Developer → Platform: distribusi
  • Developer → Ad Network: monetisasi
  • Developer → Payment: proses bayar
  • User gratis ← Advertiser bayar
  • Platform: dapat komisi transaksi
Semua saling bergantung. Sukses satu → sukses lainnya.

Flywheel Effect

Network effect → pertumbuhan makin cepat, seperti roda gila makin kencang.

Studi Kasus: Gojek

1. Banyak driver → tunggu singkat → tarik penumpang
2. Banyak penumpang → order banyak → driver untung
3. User besar → GoFood tumbuh → restoran gabung
4. GoPay integrasi → ekosistem lengkap → loyalitas naik

Tugas Riset & Diskusi Kelas

Studi Kasus: Flywheel Effect

Instruksi

Pilih satu perusahaan digital (Indonesia atau internasional) dan analisis bagaimana mereka menerapkan flywheel effect dalam ekosistem mereka.

Contoh Perusahaan

Indonesia: Tokopedia, Bukalapak, Shopee, Traveloka

Internasional: Amazon, Uber, Airbnb, Netflix

Pertanyaan Pemandu

  1. Bagaimana roda gila perusahaan ini bekerja? (Identifikasi siklus)
  2. Siapa saja pemain dalam ekosistem mereka?
  3. Model bisnis apa yang digunakan?
  4. Apa keunggulan kompetitif dari flywheel mereka?
  5. Tantangan apa yang mereka hadapi?

Pertimbangan Etis

Transparansi & Privasi

User harus sadar data dikumpulkan. Perusahaan: transparansi, jaga keamanan, beri kontrol, patuhi regulasi.

Regulasi Indonesia

UU PDP mengatur pengumpulan & perlindungan data.

Data = aset strategis. Kelola lalai → hilang trust + sanksi.

Application Programming Interface

"Lem" Ekosistem Digital

API = antarmuka komunikasi & pertukaran data antar aplikasi.

Fungsi

Lem satukan layanan digital, kolaborasi tanpa bangun semua dari nol.

Contoh Penggunaan API

Gojek × Google Maps

Navigasi tanpa bangun peta. Driver 20% lebih cepat.

Traveloka × Airline/Hotel

Data real-time ratusan maskapai tanpa simpan lokal.

OAuth Authentication

"Login with Google/Facebook" tanpa bikin akun baru.

Payment Gateway

Midtrans, Xendit, Stripe - berbagai metode bayar.

API → ekosistem kolaboratif → pertumbuhan eksponensial.

Kesimpulan

  1. Model bisnis tentukan strategi: gratis vs premium
  2. Ekosistem: developer, user, platform, advertiser, payment
  3. Network effect = advantage: yang besar makin besar
  4. Data = aset: user → data → insight → revenue
  5. API: integrasi → partnership → pertumbuhan
